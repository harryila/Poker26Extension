# A3 — Non-CoT causal-patching parity: NOT RUNNABLE (negative finding)

## Status

**A3 (non-CoT layer-sweep parity vs CoT) cannot be executed on the available
inputs.** This is itself the headline result.

## Why

A3's design pairs `clean_check_or_call` (source) against `illegal_fold`
(target) in the **non-CoT** baseline runs (`logs/scaled_*_informative_v2_enriched.jsonl.gz`).
The pre-flight count using `experiments.causal_patching.classify_decision`
on every non-CoT enriched log on disk reveals:

## What this means scientifically

This is **stronger than Hypothesis B** in `run_causal_patching_nocot_parity.sh`'s
docstring. Hypothesis B was "L\* shifts or vanishes without CoT — CoT is
reshaping internals." The actual finding is that **non-CoT mode does not
produce the failure mode at all**: every single decision is well-formed
JSON with a legal action. There is no L\* to identify because there is
no failure to mediate.

Implication for the paper:
- The deliberation-circuit story at L\*=14 (Llama), L=14-16 (Ministral),
  L=19-23 (Qwen) is conditional on CoT. CoT generates the multi-token
  reasoning trace within which the model commits the illegal-fold verb
  (typically because the trace overrides the tactical `clean_check_or_call`
  prior with a deliberative `FOLD` token).
- One-shot non-CoT outputs the verb in one step from the same prompt-prior
  and never selects the illegal verb.
- Therefore the L\* circuit we identify is best read as a **CoT-induced
  deliberation circuit**, not the model's universal action-decision
  circuit.

## Why we did not regenerate the non-CoT logs

The runbook's alternative was to run `scripts/run_tier1a_small.sh` (~3-6 h
of compute) to regenerate non-CoT logs and try again. The audit above
shows this would not help: every non-CoT condition we have, including
two temperatures, produces n=0 illegal_folds. Regenerating with the same
sampling distribution would reproduce the same outcome, at a cost of
3-6 h of GPU time.

## What the script writes vs what we have

The driver script `scripts/run_causal_patching_nocot_parity.sh` correctly
auto-skips each cell with the message:
```
[skip] <model>: only 0 illegal_FOLDs in non-CoT log
       (< 3 minimum). Non-CoT runs typically have far fewer
       illegal_FOLDs because the verb is committed in one step.
```

This SUMMARY.md and the accompanying `illegal_fold_audit.txt` document
that the skip is **complete** (across all 18 logs, not just s42), and
that the appropriate paper-level interpretation is the negative
hypothesis-B result above.
